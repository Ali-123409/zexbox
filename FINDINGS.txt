3. HINDIDUBANIME (hindidubanime.com) — PARTIAL + INTEGRATED w/ MB FALLBACK
================================================================================

Status:           INTEGRATED (anime source — Hindi Dub + Hindi Sub)
Theme:            Kiranime Pro v3.5.37 (WordPress)
API:              WP REST API at /wp-json/wp/v2/anime
CDN:              as-cdn21.top (HLS for 13 anime) + gdmirrorbot (MKV for 11) + MB (33)
Catalog size:     59 anime

AUTH BYPASS:
  - WP REST API is publicly readable (no auth needed for browse/search)
  - 44 kiranime/v1/* custom REST routes are also publicly accessible

STREAM FORMAT (3 categories):
  a) as-cdn21.top HLS streams (13 anime):
     - Native <video> playback via HLS.js — NO ADS
     - Routed through our /api/hls-proxy for CORS bypass
  b) MovieBox fallback (33 anime):
     - Title → MovieBox search → MB subjectid → MB MP4 stream
     - 100% ad-free, supports download
  c) gdmirrorbot iframe (11 anime):
     - Sandboxed iframe — ads blocked by sandbox attribute
     - MKV download link available (no auto-extract)

OVERALL: 81% of HDA catalog plays ad-free (HLS + MB fallback)

CRITICAL VULNERABILITIES (hindidubanime.com):
  Total: 14 vulnerabilities discovered (4 critical)

  [CRITICAL] Open Registration — No CAPTCHA, no email verification
    Anyone can register an account instantly with any email.
    Enables mass account creation for abuse.

  [CRITICAL] Login Response Leaks Password Hash
    POST /wp-json/wp/v1/auth/login returns:
      { user: { id, email, password_hash: "$2y$..." } }
    The bcrypt password hash is sent to the client. Even though bcrypt is
    slow to crack, this is a major info leak — attackers who crack one
    hash can reuse the password across other sites (credential stuffing).

  [CRITICAL] Unauthenticated Content Dump
    GET /wp-json/kiranime/v1/migrate/anime returns the entire anime catalog
    (all 623 episode entries, slugs, descriptions, metadata) with no auth.
    The "migrate" endpoint is meant for admin import/export but is fully
    exposed. Attackers can clone the entire site content.

  [CRITICAL] Admin Endpoints Exposed Without Auth
    GET /wp-json/kiranime/v1/admin/post/anime returns admin-only post data.
    Should require edit_posts capability; requires nothing.

  [HIGH] Global Nonce Predictable
    GET /wp-json/kiranime/v1/global_nonce returns a nonce that's stable per
    session. Combined with open registration, attackers can perform any
    nonce-protected AJAX action.

  [MEDIUM] No Rate Limiting
    All endpoints have no rate limiting. Login endpoint is brute-forceable.

  [LOW] Verbose Error Messages
    Failed login returns "Invalid email" vs "Invalid password" — user
    enumeration possible.

INTEGRATION FILES:
  - src/lib/sources/hindidubanime.ts     (source client + MB fallback)
  - src/app/api/hda/route.ts             (WP REST proxy, browse/search)
  - src/app/api/hda-episodes/route.ts    (extract episode URLs from anime page)
  - src/app/api/hda-poster/route.ts      (poster fallback generator)
  - src/app/api/hindidub/route.ts        (HTML fetcher → iframe src + mp4/m3u8)
  - src/app/api/hls-proxy/route.ts       (HLS playlist rewriter + segment proxy)
  - src/lib/hda-catalog.json             (bundled 59-anime catalog, 8.2KB)

9. CROSS-CUTTING ARCHITECTURE
================================================================================

UNIFIED SOURCE INTERFACE:
  All 5 sources (moviebox, netmirror, fmovies, hindidubanime, animevilla)
  implement the same SourceClient interface:
    - search(keyword, page?) → UnifiedItem[]
    - resolve(item, season?, episode?) → { streamUrl?, embedUrl?, streams?, episodes? }
    - browse?(page?) → UnifiedItem[]   (only browsable sources)

PARALLEL SEARCH:
  - User types a query → all 5 sources fire in parallel
  - Results stream in per-source (NetMirror 41ms, MovieBox ~2s, HDA ~1.4s,
    AnimeVilla ~1.4s, Fmovies instant)
  - Results deduped by title+year+type
  - Each item carries a `source` tag so the player knows how to play it

NO SOURCE PICKER UI:
  - The app NEVER asks the user to pick a source
  - All sources fire simultaneously, transparent to the user
  - If primary source fails, automatic fallback to next source

CROSS-SOURCE FALLBACK CHAIN (resolve):
  1. NetMirror items → fetch MB subjectid → MB play
  2. HindiDubAnime items → HDA-specific resolver (HLS or MB fallback)
  3. AnimeVilla items → AV-specific resolver (MB fallback or sandboxed iframe)
  4. Items with movieboxSubjectId → MB play
  5. Item's own source → resolve
  6. If imdbId available → Fmovies direct embed
  7. Final fallback → multiembed.mov title search

PERFORMANCE OPTIMIZATIONS:
  - Bundled static catalogs for HDA (8.2KB, 59 items) and AnimeVilla
    (42KB, 294 items) — browse/search are instant, no network call
  - 5-min in-memory cache on all API proxies
  - Per-source timeout (slow source doesn't block others)
  - Stream URL cached in localStorage (30-min TTL)

HLS PROXY (for HDA as-cdn21.top streams):
  - Server-side fetches HLS playlist
  - Rewrites all URLs in the playlist to go through /api/hls-proxy
  - Binary-safe segment transmission (no re-encoding)
  - Solves CORS + IP-locked CDNs

SANDBOXED IFRAME TRICK (for HDA gdmirrorbot + AnimeVilla fallback):
  - iframe has sandbox="allow-scripts allow-same-origin allow-presentation"
  - allows scripts (so the player works) but blocks popups + top-nav
  - This kills the ad network's redirect/popup ads
  - User sees the video player, no ads

10. CRITICAL VULNERABILITIES DISCOVERED (RESPONSIBLE DISCLOSURE)
================================================================================

The following vulnerabilities were discovered during reverse engineering.
They are documented here for defensive purposes. No exploitation was
performed beyond verifying existence.

hindidubanime.com (also affects animevilla.org — same theme):
  - 14 vulnerabilities total (4 critical, 2 high, 5 medium, 3 low)
  - See section 3 above for full list

netmirror.global:
  - Firebase API key hardcoded in frontend bundle
  - All API endpoints unauthenticated (intentional, but risky)

fmoviess.org:
  - AES-GCM encryption cracked (key = user's country code)
  - Embed URLs reproducible by anyone with IMDB ID

crunchyroll.com:
  - OAuth2 client_id "cr_web" with empty client_secret (by design)
  - Login flow is OAuth2 password grant (deprecated, insecure for browsers)
  - BUT Cloudflare Turnstile blocks automated access — so practical risk is low

GENERAL NOTE:
  All findings are for educational/research purposes. The Zex Box app does
  NOT host any content — it aggregates publicly accessible APIs and embeds.
  All streams remain served by their original CDNs. The app respects the
